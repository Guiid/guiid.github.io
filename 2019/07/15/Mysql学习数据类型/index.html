mysqk学习数据类型

Posted on 2019-07-15

字段类型

一个sql表由两部分组成，一部分是表结构、一部分是比表内容。建立表结构就是写表列功能，告诉数据库这一列应该放什么类型的数据，放的内容叫什么。表内容就是往表中填充数据。

简单的举例就像excel表格一样。我们建立excel的时候会选择适合我们需要的表格数量，如果太多就浪费掉。同样，建立sql表的时候我们也需要选择我们适合的空间，如果太多也是会浪费掉储存空间，太少就会导致不够用。所以我们建表的原则是够用，不浪费。

sql中的内容也是数据类型的，如果上面表格中编号是整形、姓名是字符型、出生日期是日期型，一张sql表大概也是这三类组成。

他们不同之处在于占据空间于储存范围。

Tinyint默认储存是-128-127，unsigend：无符号类型，影响储存范围；M，代表宽度，(zerofill时才有意义)；zerofill：零填充。

浮点型有三种：FLOAT、DOUBLE和DECIMAL;

声明整形的时候后面加上UNSIGEND可以限制取值范围不包括负数。

字符串类型，mysql跟php一样，也能识别字符串类型，mysql可以用字符串形式来保存图片或者音频。mysql中的字符串也是单引号与双引号包含起来的。

日期和时间类

时间与日期格式有五种分别是DATE、TIME、DATETIME、TIMESTAMP、YEAR

每个时间与日期都有一个零值。当插入非法字符的时候就用零添加。另外也可以使用整形列类储存unix时间戳，代替日期与时间。

字符串类型

字符串类型分为大三类

  1. 普通小段文本，char、varchar，char是固定类型的如果内容不够时则用空白填充，后面要跟字符串长度，varchar是可变长度，也就是内容会跟随内容自动增缩。
  2. TEXT和BLOB类型，次类型大小可以改变适用于长文本或者二级制数据，BLOB适合图片或者音频。
  3. SET与ENUM，这两个都是特殊的类型，SET类型用来指定列中的值必须来自一个特定集合中的特定值。
NULL

null表示没有值或者未知的值，但不表示为空。null是不能进行运算的，因为认识值运算都i是null值。

字段属性

新建表的是仅仅是字段名、字段类型还不够，还需要增加一个字段属性，比如自动增加、自动补0和默认的设置等；

UNSIGNED

该属性只能用作数值类型，被它修饰的字段名不能为负数。

NOT NULL

表示表中的所有行的次属性必须有一个值，如果没有该行则是null。

AUTO_INCREMENT

表示如果再插入行的时候没有值，则自动产生一个唯一的标识符。

ZEROFILL

当设置次属性时候则空余部分使用0来填充，如果设置了次类型则自动成为UNSIGED类型。
